# 代码性能指标

### 1. 健壮性 

```text
* 避免和程序无关的因素，导致代码报错
* 快速给出问题的反馈
* 不会因为非必要性bug，导致程序执行中断

参数做好默认值，做好检测（例如：typeScipt）
程序做好意外情况处理

使用typeof判断参数类型是否正确。参数错误 throw new Error("错误信息：xxx")
如果参数是对象，先判断是否有传入此参数，再设置默认值，也可以将传入对象与默认对象合并

使用instanceof判断是否为某个类的实例化对象

如果有错误阻断页面，使用try catch来处理错误

如果页面中存在异步显示的字段：
	1、设置默认值
	2、父 && 子 && 孙
```

### 2. 可读性

``` 
* 代码结构清晰
* 他人能够看懂你的意思

- 优化丑陋的结构（if嵌套） 
	策略模式，状态模式 ***（设计模式）
	善于使用 && 
	回调地狱=》 
		promise,async
- 遵循命名规范
	1、量全大写
	2、类名首字母大写
	3、普通变量和方法 （小驼峰）
	4、局部变量下划线开头（_xxx）
	5、语义化
- 注释

```

### 3. 可复用性

```
* 代码就像积木一样，能够很方便的哪里需要去哪里
	重复的操作不用写第二遍
	减少代码体积
* 桥接模式
* 享元模式
* 函数式编程
* 低耦合高内聚
```

### 4. 可扩展性

```
代码应对变更能力，不需要大规模改动
* 增加新需求，不用改变现有结构
* 增加新功能，只需要写新功能的代码就好
 
 
 留出扩展接口（类库extends）
 模块低耦合，高内聚
 良好的模块组织
 良好的扩展方案
 
```
