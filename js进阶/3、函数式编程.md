# 函数式编程 （规范）

### 编程规范

```
1、 面向过程
	按照步骤一步一步走
2、 面向对象
	功能组织成对象，根据相关操作作为对象的方法
	优点:
		擅长组织逻辑
3、 函数式编程（提高复用性和可扩展性）
	功能分解为一系列独立的函数，通过函数间互相调用来完成功能
	优点：
		可复用性与扩展性高
```



### 为什么前端要使用函数式编程

```text
1、可复用性
2、可拓展性
3、更好的利用Tree-shaking（webpack—树摇） 
	Tree-shaking 可以过滤多余代码（无引用变量，将不会打包进最终的包里）
	实际情况中，虽然依赖了某个模块，但其实只使用其中的某些功能。通过 tree-shaking，将没有使用的模块摇掉，这样来达到删除无用代码的目的。
```
